Certificate Renewal & Security Upgrades for Jenkins / Nexus
Problem

Security-critical services like Jenkins and Nexus rely on TLS/SSL certificates that expire and must be renewed or upgraded. If handled carelessly the service can go down, stay on an outdated certificate, or — worst case — lose data during the change.

What I did

I generated new certificates and transferred them to the target server hosting the Jenkins/Nexus setup, then created and configured SSH keys for secure access to carry out the remaining steps. I applied the certificate upgrade carefully — taking a cautious, backup-minded approach to avoid any data loss — and verified the outcome: the application came back to a normal running state and the certificate showed an updated validity date.
